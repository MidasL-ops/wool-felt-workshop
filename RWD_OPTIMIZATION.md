## 🎯 RWD 實作方式

### 1. 使用 Tailwind CSS 響應式斷點

網站使用 Tailwind CSS 的響應式類別，支援以下斷點：

- **sm:** 640px 以上（小螢幕）
- **md:** 768px 以上（平板）
- **lg:** 1024px 以上（桌面）
- **xl:** 1280px 以上（大桌面）

### 2. 響應式布局範例

#### 網格布局
```tsx
// 手機：1 欄，平板：2 欄，桌面：3 欄
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-8">
```

#### 字體大小
```tsx
// 手機：4xl，平板：5xl，桌面：6xl
<h1 className="text-4xl md:text-5xl lg:text-6xl">
```

#### 間距
```tsx
// 手機：px-4，平板：px-6，桌面：px-8
<div className="px-4 sm:px-6 lg:px-8">
```

## 📐 響應式斷點標準

| 裝置類型 | 螢幕寬度 | Tailwind 斷點 | 布局特點 |
|---------|---------|-------------|---------|
| 手機 | < 640px | 預設 | 單欄，垂直堆疊 |
| 小平板 | 640px - 768px | `sm:` | 開始使用兩欄 |
| 平板 | 768px - 1024px | `md:` | 兩欄布局 |
| 桌面 | 1024px - 1280px | `lg:` | 三欄布局 |
| 大桌面 | > 1280px | `xl:` | 四欄布局 |

---

## 🎨 響應式設計原則

### 1. Mobile First（手機優先）
- 所有樣式預設為手機版
- 使用 `md:`, `lg:` 等類別逐步增強

### 2. 彈性圖片
- 使用 `w-full` 和 `object-cover` 讓圖片自適應
- Next.js Image 元件自動優化

### 3. 觸控友善
- 按鈕最小尺寸 44x44px
- 足夠的間距避免誤觸

### 4. 可讀性
- 字體大小隨螢幕調整
- 適當的行高和間距
